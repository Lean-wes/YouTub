📄 Fundamentos de HTML

+

Introducción a HTML

Nivel:

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. Define la estructura básica de una página web mediante una serie de elementos o etiquetas.

Estructura básica de un documento HTML:

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Título de la página</title>
</head>
<body>
    <h1>Este es un encabezado</h1>
    <p>Este es un párrafo.</p>
</body>
</html>

Elementos HTML

Nivel:

Los elementos HTML están definidos por etiquetas. Las etiquetas vienen en pares: una etiqueta de apertura y una de cierre.

Encabezados

HTML ofrece seis niveles de encabezados, de h1 a h6:

<h1>Encabezado 1</h1>
<h2>Encabezado 2</h2>
<h3>Encabezado 3</h3>
<h4>Encabezado 4</h4>
<h5>Encabezado 5</h5>
<h6>Encabezado 6</h6>

Párrafos

<p>Este es un párrafo.</p>

Enlaces

<a href="https://www.ejemplo.com">Visita nuestro sitio</a>

Imágenes

<img src="imagen.jpg" alt="Descripción de la imagen">

Listas

Listas ordenadas:

<ol>
    <li>Primer elemento</li>
    <li>Segundo elemento</li>
    <li>Tercer elemento</li>
</ol>

Listas no ordenadas:

<ul>
    <li>Elemento</li>
    <li>Otro elemento</li>
    <li>Un elemento más</li>
</ul>

🎨 CSS: Estilizando la Web

+

Introducción a CSS

Nivel:

CSS (Cascading Style Sheets) es el lenguaje utilizado para describir la presentación de documentos HTML. Permite controlar el diseño, colores, fuentes y disposición de los elementos en una página web.

Formas de incluir CSS

1. CSS Interno (en el head del documento HTML):

<head>
    <style>
        body {
            background-color: lightblue;
        }
        h1 {
            color: navy;
            margin-left: 20px;
        }
    </style>
</head>

2. CSS Externo (en un archivo separado):

<head>
    <link rel="stylesheet" href="styles.css">
</head>

3. CSS en línea (directamente en el elemento HTML):

<h1 style="color: blue; margin-left: 30px;">Título</h1>

Selectores CSS

Nivel:

Los selectores CSS permiten seleccionar y modificar elementos HTML específicos:

Selector de elemento

p {
    color: red;
    text-align: center;
}

Selector de ID

#mi-id {
    background-color: lightblue;
}

Selector de clase

.mi-clase {
    font-size: 16px;
    margin: 10px;
}

Selectores combinados

div p {
    /* Selecciona todos los párrafos dentro de divs */
    color: blue;
}

.container > p {
    /* Selecciona párrafos que son hijos directos de elementos con clase container */
    font-weight: bold;
}

h2 + p {
    /* Selecciona párrafos que son adyacentes directos a h2 */
    color: green;
}

a[href^="https"] {
    /* Selecciona enlaces que comienzan con https */
    color: purple;
}

📱 JavaScript: Programación del Lado del Cliente

+

Introducción a JavaScript

Nivel:

JavaScript es un lenguaje de programación que permite implementar funciones complejas en páginas web. Cada vez que una página web hace más que simplemente mostrar información estática (como mostrar contenido actualizado, mapas interactivos, animaciones, etc.), JavaScript probablemente está involucrado.

Cómo incluir JavaScript

<!-- Interno -->
<script>
    console.log("Hola mundo");
</script>

<!-- Externo -->
<script src="script.js"></script>

Variables y Tipos de Datos

// Variables con let y const
let nombre = "María"; // String
const edad = 30;      // Number
let esMayor = true;   // Boolean
let colores = ["rojo", "verde", "azul"]; // Array
let persona = {       // Object
    nombre: "Juan",
    edad: 25,
    profesion: "Desarrollador"
};

Funciones

// Función declarativa
function saludar(nombre) {
    return `Hola, ${nombre}!`;
}

// Función de expresión
const despedir = function(nombre) {
    return `Adiós, ${nombre}!`;
};

// Función de flecha
const multiplicar = (a, b) => a * b;

DOM (Document Object Model)

Nivel:

El DOM es una representación en memoria de un documento HTML que permite a JavaScript acceder y manipular el contenido, estructura y estilos de una página web.

Seleccionar elementos

// Por ID
const elemento = document.getElementById('mi-id');

// Por clase
const elementos = document.getElementsByClassName('mi-clase');

// Por etiqueta
const parrafos = document.getElementsByTagName('p');

// Con selectores CSS (devuelve el primer elemento que coincide)
const primerElemento = document.querySelector('.container p');

// Con selectores CSS (devuelve todos los elementos que coinciden)
const todosElementos = document.querySelectorAll('.container p');

Modificar elementos

// Cambiar contenido
elemento.textContent = 'Nuevo texto';
elemento.innerHTML = '<strong>Texto en negrita</strong>';

// Cambiar atributos
elemento.setAttribute('src', 'nueva-imagen.jpg');
elemento.id = 'nuevo-id';

// Cambiar estilos
elemento.style.color = 'red';
elemento.style.fontSize = '16px';

// Añadir/eliminar clases
elemento.classList.add('clase-nueva');
elemento.classList.remove('clase-antigua');
elemento.classList.toggle('clase-toggle');

Crear y añadir elementos

// Crear elemento
const nuevoElemento = document.createElement('div');
nuevoElemento.textContent = 'Soy un nuevo elemento';
nuevoElemento.className = 'mi-clase';

// Añadir elemento al DOM
document.body.appendChild(nuevoElemento);

// Insertar antes de otro elemento
const elementoReferencia = document.getElementById('referencia');
elementoReferencia.parentNode.insertBefore(nuevoElemento, elementoReferencia);

🖥️ Desarrollo Backend

+

Introducción al Backend

Nivel:

El desarrollo backend se refiere a la parte del servidor de una aplicación web. Se encarga de la lógica de negocio, almacenamiento de datos, seguridad y comunicación con el frontend.

Lenguajes y Frameworks

  • Node.js: Entorno de ejecución para JavaScript en el servidor (Express.js, Nest.js)
  • Python: Lenguaje versátil y legible (Django, Flask)
  • PHP: Lenguaje específico para web (Laravel, Symfony)
  • Java: Lenguaje robusto y empresarial (Spring Boot)
  • Ruby: Lenguaje elegante y productivo (Ruby on Rails)
  • Go: Lenguaje eficiente y de alto rendimiento
  • C#: Lenguaje de Microsoft (.NET Core)

APIs RESTful

Nivel:

Una API (Interfaz de Programación de Aplicaciones) RESTful es un estilo arquitectónico para diseñar servicios web que utiliza HTTP como protocolo de comunicación y sigue principios específicos.

Principios REST

  • Recursos: Todo es un recurso representado por URIs
  • Métodos HTTP: Utiliza métodos estándar (GET, POST, PUT, DELETE)
  • Sin estado: Cada solicitud debe contener toda la información necesaria
  • Representación: Los recursos pueden tener múltiples representaciones (JSON, XML)
  • HATEOAS: Hypermedia como motor del estado de la aplicación

Métodos HTTP comunes

  • GET: Recuperar datos
  • POST: Crear un nuevo recurso
  • PUT: Actualizar un recurso existente
  • DELETE: Eliminar un recurso
  • PATCH: Actualizar parcialmente un recurso

Ejemplo de API RESTful (Express.js)

const express = require('express');
const app = express();
app.use(express.json());

let usuarios = [
    { id: 1, nombre: 'Ana', edad: 28 },
    { id: 2, nombre: 'Carlos', edad: 34 }
];

// GET - Obtener todos los usuarios
app.get('/api/usuarios', (req, res) => {
    res.json(usuarios);
});

// GET - Obtener un usuario específico
app.get('/api/usuarios/:id', (req, res) => {
    const id = parseInt(req.params.id);
    const usuario = usuarios.find(u => u.id === id);
    
    if (!usuario) {
        return res.status(404).json({ mensaje: 'Usuario no encontrado' });
    }
    
    res.json(usuario);
});

// POST - Crear un nuevo usuario
app.post('/api/usuarios', (req, res) => {
    const nuevoUsuario = {
        id: usuarios.length + 1,
        nombre: req.body.nombre,
        edad: req.body.edad
    };
    
    usuarios.push(nuevoUsuario);
    res.status(201).json(nuevoUsuario);
});

// PUT - Actualizar un usuario
app.put('/api/usuarios/:id', (req, res) => {
    const id = parseInt(req.params.id);
    const index = usuarios.findIndex(u => u.id === id);
    
    if (index === -1) {
        return res.status(404).json({ mensaje: 'Usuario no encontrado' });
    }
    
    usuarios[index] = {
        id: id,
        nombre: req.body.nombre,
        edad: req.body.edad
    };
    
    res.json(usuarios[index]);
});

// DELETE - Eliminar un usuario
app.delete('/api/usuarios/:id', (req, res) => {
    const id = parseInt(req.params.id);
    const index = usuarios.findIndex(u => u.id === id);
    
    if (index === -1) {
        return res.status(404).json({ mensaje: 'Usuario no encontrado' });
    }
    
    const usuarioEliminado = usuarios[index];
    usuarios.splice(index, 1);
    
    res.json(usuarioEliminado);
});

app.listen(3000, () => {
    console.log('Servidor corriendo en el puerto 3000');
});

🗄️ Bases de Datos

+

Bases de Datos Relacionales (SQL)

Nivel:

Las bases de datos relacionales organizan los datos en tablas con filas y columnas, estableciendo relaciones entre ellas mediante claves.

Sistemas Populares

  • MySQL: De código abierto, muy utilizado en aplicaciones web
  • PostgreSQL: Potente, de código abierto con características avanzadas
  • SQLite: Base de datos ligera basada en archivos
  • SQL Server: Solución empresarial de Microsoft
  • Oracle: Sistema robusto para aplicaciones empresariales

Comandos SQL Básicos

-- Crear una tabla
CREATE TABLE usuarios (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    fecha_registro DATE DEFAULT CURRENT_DATE
);

-- Insertar datos
INSERT INTO usuarios (nombre, email) 
VALUES ('Juan Pérez', 'juan@ejemplo.com');

-- Consultar datos
SELECT * FROM usuarios;
SELECT nombre, email FROM usuarios WHERE id = 1;

-- Actualizar datos
UPDATE usuarios SET nombre = 'Juan Carlos Pérez' WHERE id = 1;

-- Eliminar datos
DELETE FROM usuarios WHERE id = 1;

-- Joins (uniones)
SELECT usuarios.nombre, pedidos.producto
FROM usuarios
JOIN pedidos ON usuarios.id = pedidos.usuario_id;

Bases de Datos NoSQL

Nivel:

Las bases de datos NoSQL están diseñadas para modelos de datos específicos y tienen esquemas flexibles, lo que las hace ideales para datos no estructurados o semiestructurados.

Tipos de Bases de Datos NoSQL

  • Documentales: Almacenan datos en documentos similares a JSON (MongoDB)
  • Clave-valor: Almacenan pares de clave-valor (Redis, DynamoDB)
  • Columnares: Optimizadas para consultas en grandes conjuntos de datos (Cassandra)
  • Grafo: Almacenan relaciones entre entidades (Neo4j)

Ejemplo con MongoDB

// Conectar a MongoDB
const { MongoClient } = require('mongodb');
const uri = "mongodb://localhost:27017";
const client = new MongoClient(uri);

async function main() {
    try {
        await client.connect();
        const database = client.db("miApp");
        const usuarios = database.collection("usuarios");
        
        // Insertar un documento
        const resultado = await usuarios.insertOne({
            nombre: "Ana García",
            email: "ana@ejemplo.com",
            edad: 28,
            intereses: ["programación", "música", "viajes"]
        });
        console.log(`Documento insertado con ID: ${resultado.insertedId}`);
        
        // Buscar documentos
        const consulta = { edad: { $gt: 25 } }; // usuarios mayores de 25 años
        const usuario = await usuarios.findOne(consulta);
        console.log(usuario);
        
        // Actualizar un documento
        const filtro = { email: "ana@ejemplo.com" };
        const actualizacion = {
            $set: { edad: 29 },
            $push: { intereses: "fotografía" }
        };
        await usuarios.updateOne(filtro, actualizacion);
        
        // Eliminar un documento
        await usuarios.deleteOne({ email: "ana@ejemplo.com" });
        
    } finally {
        await client.close();
    }
}

main().catch(console.error);

🔒 Seguridad Web

+

Vulnerabilidades Comunes

Nivel:

La seguridad web es crucial para proteger tanto los datos de los usuarios como la integridad de la aplicación. Estas son algunas vulnerabilidades comunes y cómo prevenirlas:

1. Inyección SQL

Problema: Los atacantes pueden insertar código SQL malicioso a través de entradas no validadas.

Prevención: Usar consultas parametrizadas o ORM.

// Vulnerable
db.query(`SELECT * FROM usuarios WHERE email = '${email}'`);

// Seguro (consulta parametrizada)
db.query('SELECT * FROM usuarios WHERE email = ?', [email]);

2. Cross-Site Scripting (XSS)

Problema: Los atacantes pueden inyectar scripts maliciosos que se ejecutan en los navegadores de los usuarios.

Prevención: Escapar o sanitizar la entrada del usuario antes de mostrarla.

// Vulnerable
element.innerHTML = userInput;

// Seguro
element.textContent = userInput; // o usar una biblioteca de sanitización

3. Cross-Site Request Forgery (CSRF)

Problema: Fuerza a los usuarios autenticados a ejecutar acciones no deseadas.

Prevención: Implementar tokens CSRF y validar el origen de las solicitudes.

<form action="/transferir" method="post">
    <input type="hidden" name="csrf_token" value="token_aleatorio_generado_por_servidor">
    <!-- otros campos del formulario -->
    <button type="submit">Transferir</button>
</form>

4. Autenticación y Gestión de Sesiones Débiles

Problema: Métodos de autenticación inseguros que permiten el acceso no autorizado.

Prevención:

  • Implementar políticas de contraseñas fuertes
  • Usar HTTPS para todas las comunicaciones
  • Implementar autenticación multifactor
  • Usar tokens JWT con expiración adecuada
  • Revocar sesiones al cerrar sesión

5. Exposición de Datos Sensibles

Problema: Información sensible transmitida o almacenada sin cifrar.

Prevención:

  • Cifrar datos sensibles en reposo y en tránsito
  • Usar funciones hash seguras para contraseñas (bcrypt, Argon2)
  • No almacenar datos sensibles innecesariamente
// Cifrado de contraseñas con bcrypt
const bcrypt = require('bcrypt');

// Almacenar contraseña
const hashContraseña = async (contraseña) => {
    const saltRounds = 10;
    return await bcrypt.hash(contraseña, saltRounds);
};

// Verificar contraseña
const verificarContraseña = async (contraseña, hash) => {
    return await bcrypt.compare(contraseña, hash);
};

Cabeceras de Seguridad HTTP

Nivel:

Las cabeceras HTTP pueden mejorar significativamente la seguridad de tu aplicación web:

Cabeceras Importantes

// Ejemplo en Express.js
const helmet = require('helmet');
app.use(helmet()); // Añade varias cabeceras de seguridad

// O configurar manualmente
app.use((req, res, next) => {
    // Previene que el navegador MIME-sniff una respuesta de su tipo de contenido declarado
    res.setHeader('X-Content-Type-Options', 'nosniff');
    
    // Ayuda a proteger contra ataques XSS
    res.setHeader('X-XSS-Protection', '1; mode=block');
    
    // Evita que el sitio aparezca en un iframe (clickjacking)
    res.setHeader('X-Frame-Options', 'DENY');
    
    // Política de seguridad de contenido (CSP)
    res.setHeader(
        'Content-Security-Policy',
        "default-src 'self'; script-src 'self' https://trusted-cdn.com;"
    );
    
    // Strict Transport Security (fuerza HTTPS)
    res.setHeader(
        'Strict-Transport-Security',
        'max-age=31536000; includeSubDomains; preload'
    );
    
    next();
});

🚀 Despliegue y DevOps

+

Opciones de Despliegue

Nivel:

Existen múltiples formas de desplegar aplicaciones web, cada una con sus ventajas y casos de uso:

Hosting Compartido

Descripción: Varios sitios web comparten los recursos de un único servidor.

Ventajas: Económico, fácil de configurar

Desventajas: Recursos limitados, menos control, posibles problemas de rendimiento

Ejemplos: Hostinger, Bluehost, GoDaddy

VPS (Servidor Privado Virtual)

Descripción: Un servidor virtual dedicado con recursos garantizados.

Ventajas: Mayor control, recursos dedicados, mejor rendimiento

Desventajas: Requiere más conocimientos técnicos, más caro que el hosting compartido

Ejemplos: DigitalOcean, Linode, Vultr

Plataformas como Servicio (PaaS)

Descripción: Plataformas que abstraen la infraestructura y permiten centrarse en el código.

Ventajas: Fácil despliegue, escalabilidad automática, menos administración

Desventajas: Menos flexibilidad, posiblemente más caro a escala

Ejemplos: Heroku, Netlify, Vercel, Firebase

Contenedores

Descripción: Paquetes ligeros y portables que contienen todo lo necesario para ejecutar la aplicación.

Ventajas: Consistencia entre entornos, aislamiento, eficiencia de recursos

Desventajas: Curva de aprendizaje, complejidad añadida

Ejemplos: Docker, Kubernetes

Serverless

Descripción: El código se ejecuta en respuesta a eventos sin necesidad de gestionar servidores.

Ventajas: Pago por uso, escalado automático, menos mantenimiento

Desventajas: Cold starts, límites de tiempo de ejecución, posible vendor lock-in

Ejemplos: AWS Lambda, Azure Functions, Google Cloud Functions

Integración Continua y Despliegue Continuo (CI/CD)

Nivel:

CI/CD es una práctica de desarrollo que implica la integración frecuente de código y la entrega automatizada de cambios a producción.

Componentes

  • Integración Continua (CI): Verificación automática de cada cambio mediante pruebas
  • Entrega Continua (CD): Automatización del proceso de publicación para su despliegue manual
  • Despliegue Continuo: Automatización completa hasta la puesta en producción

Beneficios

  • Detección temprana de errores
  • Retroalimentación rápida
  • Reducción de riesgos en los despliegues
  • Entregas más frecuentes y predecibles
  • Menor esfuerzo manual

Ejemplo: GitHub Actions

name: Node.js CI/CD

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:

    runs-on: ubuntu-latest

    - uses: actions/checkout@v2

      uses: actions/setup-node@v2
      with:
        node-version: '14'
        
    - name: Install dependencies
      run: npm ci
      
    - name: Build
      run: npm run build
      
    - name: Deploy to production
      uses: some-deployment-action@v1
      with:
        api-key: ${ { secrets.DEPLOY_API_KEY } }
        app-name: 'my-app-production'

⚡ Rendimiento Web

+

Métricas de Rendimiento

Nivel:

Las métricas de rendimiento web son esenciales para medir la experiencia del usuario:

Métricas Clave

  • First Contentful Paint (FCP): Tiempo hasta que se muestra el primer contenido
  • Largest Contentful Paint (LCP): Tiempo hasta que se muestra el contenido principal
  • First Input Delay (FID): Tiempo hasta que la página responde a la primera interacción
  • Cumulative Layout Shift (CLS): Medida de inestabilidad visual de la página
  • Time to Interactive (TTI): Tiempo hasta que la página es completamente interactiva
  • Total Blocking Time (TBT): Tiempo total en que el hilo principal está bloqueado

Herramientas de Medición

  • Lighthouse: Auditoría automatizada de rendimiento, accesibilidad y más
  • WebPageTest: Pruebas detalladas de rendimiento desde múltiples ubicaciones
  • Chrome DevTools: Herramientas integradas en el navegador para análisis de rendimiento
  • Core Web Vitals: Métricas estandarizadas por Google (LCP, FID, CLS)

Optimización de Rendimiento

Nivel:

Técnicas para mejorar el rendimiento de aplicaciones web:

Optimización de Recursos

  • Minificación: Reducir el tamaño de archivos CSS, JavaScript y HTML
  • Compresión: Usar Gzip o Brotli para reducir el tamaño de transferencia
  • Optimización de imágenes: Usar formatos modernos (WebP, AVIF), dimensiones apropiadas y lazy loading
  • Tree shaking: Eliminar código no utilizado del paquete final
  • Code splitting: Dividir el código en paquetes más pequeños cargados bajo demanda

Optimización de Entrega

  • CDN (Content Delivery Network): Distribuir contenido en servidores cercanos al usuario
  • Caching: Implementar estrategias de caché a nivel de navegador y servidor
  • Priorización de recursos críticos: Cargar primero lo esencial para el renderizado
  • Precarga y prefetch: Indicar al navegador qué recursos cargar anticipadamente

Optimización de Renderizado

  • CSS crítico: Incluir el CSS esencial en línea y cargar el resto de forma asíncrona
  • Reducir bloqueo de renderizado: Cargar JavaScript de forma asíncrona o diferida
  • Evitar reflows y repaints: Minimizar cambios en el DOM que provoquen recálculos
  • Virtualización: Renderizar solo los elementos visibles en listas largas

Ejemplo: Carga diferida de imágenes

<!-- Usando el atributo loading nativo -->
<img src="imagen.jpg" loading="lazy" alt="Descripción">

<!-- Usando Intersection Observer (JavaScript) -->
<img data-src="imagen.jpg" class="lazy" alt="Descripción">

<script>
document.addEventListener("DOMContentLoaded", function() {
    const imageObserver = new IntersectionObserver((entries, observer) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                const image = entry.target;
                image.src = image.dataset.src;
                observer.unobserve(image);
            }
        });
    });
    
    document.querySelectorAll('img.lazy').forEach(img => {
        imageObserver.observe(img);
    });
});
</script>

🏗️ Arquitectura de Software

+

Patrones Arquitectónicos

Nivel:

Los patrones arquitectónicos son soluciones generales y reutilizables para problemas comunes en el diseño de software.

MVC (Modelo-Vista-Controlador)

Separa la aplicación en tres componentes interconectados:

  • Modelo: Representa los datos y la lógica de negocio
  • Vista: Presenta la información al usuario (interfaz)
  • Controlador: Maneja las interacciones del usuario y actualiza el modelo y la vista

MVVM (Modelo-Vista-ViewModel)

Evolución del MVC utilizada en aplicaciones con interfaces complejas:

  • Modelo: Datos y lógica de negocio
  • Vista: Interfaz de usuario
  • ViewModel: Intermediario que convierte los datos del modelo para la vista y maneja la lógica de presentación

Arquitectura de Microservicios

Estructura una aplicación como un conjunto de servicios pequeños e independientes:

  • Cada servicio se ejecuta en su propio proceso
  • Comunicación a través de APIs ligeras (generalmente HTTP/REST)
  • Despliegue independiente de cada servicio
  • Diferentes servicios pueden usar diferentes tecnologías y bases de datos
  • Escalado granular (solo los servicios que lo necesiten)

Arquitectura Serverless

Permite construir aplicaciones sin gestionar la infraestructura subyacente:

  • Las funciones se activan por eventos
  • Ejecución en entornos efímeros y sin estado
  • Escalado automático y pago por uso
  • Enfoque en la lógica de negocio en lugar de la infraestructura

Patrones de Diseño

Nivel:

Los patrones de diseño son soluciones probadas para problemas recurrentes en el diseño de software.

Patrones Creacionales

  • Singleton: Garantiza una única instancia de una clase
  • Factory Method: Define una interfaz para crear objetos, pero permite a las subclases decidir qué clase instanciar
  • Builder: Separa la construcción de objetos complejos de su representación

Patrones Estructurales

  • Adapter: Permite que interfaces incompatibles trabajen juntas
  • Decorator: Añade responsabilidades a objetos dinámicamente
  • Facade: Proporciona una interfaz unificada a un conjunto de interfaces en un subsistema

Patrones de Comportamiento

  • Observer: Define una dependencia uno-a-muchos entre objetos
  • Strategy: Define una familia de algoritmos, encapsula cada uno y los hace intercambiables
  • Command: Encapsula una solicitud como un objeto

Ejemplo: Patrón Observer (JavaScript)

class Subject {
    constructor() {
        this.observers = [];
    }
    
    subscribe(observer) {
        this.observers.push(observer);
    }
    
    unsubscribe(observer) {
        this.observers = this.observers.filter(obs => obs !== observer);
    }
    
    notify(data) {
        this.observers.forEach(observer => observer.update(data));
    }
}

class Observer {
    constructor(name) {
        this.name = name;
    }
    
    update(data) {
        console.log(`${this.name} recibió: ${data}`);
    }
}

// Uso
const subject = new Subject();

const observer1 = new Observer("Observer 1");
const observer2 = new Observer("Observer 2");

subject.subscribe(observer1);
subject.subscribe(observer2);

subject.notify("¡Datos actualizados!");
// Observer 1 recibió: ¡Datos actualizados!
// Observer 2 recibió: ¡Datos actualizados!

subject.unsubscribe(observer1);
subject.notify("Más actualizaciones");
// Observer 2 recibió: Más actualizaciones

Nivel:

TDD (Test-Driven Development)

Proceso de desarrollo que se basa en escribir pruebas antes del código:

  1. Escribir una prueba que falla para una nueva funcionalidad

Extensión de TDD que se enfoca en el comportamiento desde la perspectiva del usuario:

  • Las especificaciones se pueden automatizar como pruebas

Nivel:

// Registrar un Service Worker
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js')
      .then(registration => {
        console.log('SW registrado:', registration);
      })
      .catch(error => {
        console.log('Error al registrar SW:', error);
      });
  });
}

// sw.js - Service Worker
const CACHE_NAME = 'mi-app-cache-v1';
const urlsToCache = [
  '/',
  '/styles/main.css',
  '/scripts/main.js',
  '/images/logo.png'
];

// Instalar SW y crear caché
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => {
        return cache.addAll(urlsToCache);
      })
  );
});

// Interceptar solicitudes de red
self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => {
        // Devolver de caché si está disponible
        if (response) {
          return response;
        }
        // Si no, hacer solicitud a la red
        return fetch(event.request);
      })
  );
});

♿ Accesibilidad Web

+

Fundamentos de Accesibilidad

Nivel:

La accesibilidad web significa que personas con discapacidades pueden percibir, comprender, navegar e interactuar con la web.

Principios WCAG

Los estándares de accesibilidad web están definidos por las Pautas de Accesibilidad al Contenido Web (WCAG) y se basan en cuatro principios:

  1. Perceptible: La información debe ser presentada de manera que los usuarios puedan percibirla
  2. Operable: La interfaz no debe requerir interacciones que un usuario no pueda realizar
  3. Comprensible: La información y el funcionamiento deben ser comprensibles
  4. Robusto: El contenido debe ser suficientemente robusto para funcionar con tecnologías actuales y futuras

Niveles de Conformidad

  • Nivel A: Nivel mínimo de accesibilidad
  • Nivel AA: Elimina importantes barreras de acceso (el estándar común)
  • Nivel AAA: Nivel óptimo de accesibilidad

Implementación de la Accesibilidad

Nivel:

Técnicas prácticas para hacer que los sitios web sean accesibles:

Semántica HTML

Usar elementos HTML apropiados según su significado.

<!-- Mal ejemplo -->
<div class="header">Mi Sitio</div>
<div class="navigation">...</div>
<div class="main">...</div>
<div class="footer">...</div>

<!-- Buen ejemplo -->
<header>Mi Sitio</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

Atributos ARIA

ARIA (Accessible Rich Internet Applications) proporciona atributos para mejorar la accesibilidad.

<!-- Roles -->
<div role="alert">Este es un mensaje importante</div>

<!-- Estados y propiedades -->
<button aria-expanded="false" aria-controls="menu">Menú</button>
<div id="menu" aria-hidden="true">...</div>

<!-- Etiquetas -->
<input type="text" aria-label="Buscar en el sitio">
<button aria-labelledby="tooltip-1">?</button>
<div id="tooltip-1">Ayuda contextual</div>

Teclado y Foco

  • Todos los controles interactivos deben ser accesibles mediante teclado
  • El orden de tabulación debe ser lógico
  • El foco debe ser visible en todo momento
  • No usar tabindex mayor que 0 (altera el orden natural)

Contraste y Texto

  • Nivel AA: Contraste de texto mínimo 4.5:1 (3:1 para texto grande)
  • Permitir el redimensionamiento del texto hasta un 200% sin pérdida de contenido
  • No usar solo el color para transmitir información

Imágenes y Multimedia

  • Proporcionar texto alternativo para imágenes
  • Incluir subtítulos y transcripciones para audio
  • Evitar contenido que parpadee más de 3 veces por segundo
<!-- Imagen informativa -->
<img src="logo.png" alt="Logo de Mi Empresa">

<!-- Imagen decorativa -->
<img src="decoracion.png" alt="">

<!-- Video con subtítulos -->
<video controls>
    <source src="video.mp4" type="video/mp4">
    <track src="subtitulos.vtt" kind="subtitles" srclang="es" label="Español">
    Tu navegador no soporta el elemento video.
</video>

Nivel:

Herramientas y métodos para evaluar la accesibilidad de un sitio web:

Comprobación Automática

  • Lighthouse: Herramienta integrada en Chrome DevTools
  • axe: Motor de pruebas de accesibilidad integrable
  • WAVE: Herramienta web y extensión de navegador

  • Verificar el contraste de colores
  • Comprobar el comportamiento al ampliar la página (zoom)

🌐 Internet e Infraestructura Web

+

Cómo Funciona Internet

Nivel:

Fundamentos técnicos sobre cómo funciona Internet y la web.

Modelo TCP/IP

El conjunto de protocolos que gobierna cómo fluyen los datos en Internet:

  1. Capa de Aplicación: HTTP, FTP, SMTP, DNS
  2. Capa de Transporte: TCP, UDP
  3. Capa de Internet: IP, ICMP
  4. Capa de Enlace: Ethernet, Wi-Fi

DNS (Domain Name System)

Sistema que traduce nombres de dominio a direcciones IP:

  • Solicitud DNS: El navegador pregunta "¿Cuál es la IP de example.com?"
  • Resolución: Servidor DNS responde con "192.0.2.1"
  • Conexión: El navegador se conecta a 192.0.2.1

HTTP (Hypertext Transfer Protocol)

Protocolo para transferir hipertexto entre sistemas:

  • Métodos: GET, POST, PUT, DELETE, etc.
  • Códigos de estado: 200 (OK), 404 (Not Found), 500 (Server Error)
  • Cabeceras: Metadatos de la solicitud/respuesta
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Date: Mon, 18 May 2025 10:33:27 GMT

<!DOCTYPE html>
<html>...</html>

HTTPS y TLS

Versión segura de HTTP que cifra las comunicaciones:

  1. El cliente solicita una conexión segura
  2. El servidor envía su certificado
  3. El cliente verifica el certificado
  4. Ambos acuerdan una clave de sesión
  5. La comunicación queda cifrada

Servidores y Hosting

Nivel:

Infraestructura que hace que los sitios web estén disponibles en Internet.

Tipos de Servidores Web

  • Apache: Popular, maduro y extensible
  • Nginx: Alto rendimiento, especializado en contenido estático y proxy
  • IIS: Servidor web de Microsoft para Windows
  • Node.js: Entorno de ejecución que puede servir aplicaciones web

Configuración Básica de Nginx

server {
    listen 80;
    server_name example.com www.example.com;
    
    location / {
        root /var/www/html;
        index index.html index.htm;
        try_files $uri $uri/ =404;
    }
    
    location /api {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    error_page 404 /404.html;
    location = /404.html {
        root /var/www/html;
    }
}

Registradores de Dominio y DNS

Para hacer visible un sitio en Internet:

  1. Registrar un dominio en un registrador (GoDaddy, Namecheap, Google Domains)
  2. Configurar registros DNS para apuntar a tu servidor:
    • A Record: Dominio → IP
    • CNAME: Alias para otro dominio
    • MX: Servidor de correo
    • TXT: Información textual (verificación)

CDN y Escalabilidad

Nivel:

Técnicas para servir contenido eficientemente a escala global.

CDN (Content Delivery Network)

Red de servidores distribuidos geográficamente para entregar contenido:

  • Mejor rendimiento: Sirve desde el punto más cercano al usuario
  • Menor carga en el servidor de origen: La mayoría del tráfico lo maneja el CDN
  • Alta disponibilidad: Redundancia geográfica
  • Protección DDoS: Muchos CDN ofrecen escudo contra ataques

Proveedores populares: Cloudflare, Akamai, AWS CloudFront, Fastly

Escalabilidad

Escalabilidad Vertical: Aumentar la potencia del servidor

  • Más CPU, RAM, capacidad de disco
  • Límites físicos y de costo
  • Punto único de fallo

Escalabilidad Horizontal: Aumentar el número de servidores

  • Añadir más máquinas al sistema
  • Requiere balanceo de carga
  • Mayor tolerancia a fallos
  • Más fácil de escalar según demanda

Balanceo de Carga

Distribuye el tráfico entre múltiples servidores:

  • Round Robin: Distribuye secuencialmente
  • Least Connections: Envía a servidor con menos conexiones
  • IP Hash: Consistencia por IP del cliente
  • Geolocalización: Dirige a servidores más cercanos
